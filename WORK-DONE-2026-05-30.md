## Phase A — SQLite → Supabase Postgres

### Decisions
- D1: Supabase = **managed Postgres only**. Kept Prisma, NextAuth, RBAC, route handlers (no Supabase Auth/RLS).
- Connect via the Supabase **pooler** (`aws-0-eu-west-1.pooler.supabase.com`): port `6543` (transaction, runtime `DATABASE_URL`), port `5432` (session, `DIRECT_URL` for DDL). The direct `db.<ref>.supabase.co` host is IPv6-only → unreachable on IPv4; **must** use the pooler.
- DB password contains special chars → **percent-encoded** in the connection strings (`( @ )` → `%28 %40 %29`).

### Security fix
The middleware page-level auth guard had **never actually fired** — `PUBLIC_PATHS` contained `'/'` and `isPublicPath` used `path.startsWith(p)`, so every path matched `startsWith('/')` and was treated public. Root now matches **exactly**, so `/dashboard` is properly guarded server-side (`307 → /login`).
